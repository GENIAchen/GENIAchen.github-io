loc_4011A0 处，通过指针引用了一个函数，查看汇编发现无法F5，我们将标红处 nop 后重新定义函数，即可查看其伪C：

查看最重要的 sub_401830：

一直到 while 循环，一直是在对 byte_416050 的值进行处理。

我们从下往上看，因为要打印出format数组中的内容，必须在 if 语句内就跳出 while 循环。因此 sub_401830 的最后 v13 需要等于 0xAB94。这就不得不看看 sub_401470 里是什么了：

当 a2 满足所有 if 的条件后，经过一通或运算和与运算才能得到 0xAB94，而需要满足这些条件，a2 字符串必须等于：dbappsec ，注意中间有个反调试，所以索引 5 处应该是 s 。即函数外的 v16 为 dbappsec。

v16 是 byte_416050 与 v14 进行异或得到的结果。盲猜一手，v14 里存的就是我们输进去的密码。

现在就相当于知道了 byte_416050 就知道了密码。
